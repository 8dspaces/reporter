Change log for version 0.9

Summery: Added two new engines (DB and windows-eventlog). Added version validation checks. Encapsulated start-engine validation in the engine's class body. Improved classes cross reference. You can add a filter to multiple engines at once. Minor bug fixes and improvements. Improved documentation and built-in comments.

 

1.      ReporterManager

1.      Cross class linkage and knowledge transfer

1.      ReporterManager now exposes the reporters collection via Reporters read-only property

2.      Added public properties MinimumEngineVersion and MinimumFilterVersion

2.      StartEngine checks are now done from within the engine's class, not the reporter-manager

1.      AllowCreate function was removed

3.      You can add a filter to multiple engines at once:

.AddFilter("FilterType", "ReporterName1>ReporterName2>Ö.", "FilerSettings")

Please note ñ if the reporter name contains that character ' > ', double it.

E.g. .AddFilter("FilterType", "Name1>Reporter>>Name2>", "FilerSettings")

††††††††††††††††††††††† The command sets the filter for 2 engines: "Name1" and "Reporter>Name2".

Backward compatible (i.e. the command still works with only 1 engine)

††††††††††††††††††††††††††††††††††† Created AddFilter (public) which breaks the reporter names, and calls AddFilterToReporter for each one.
